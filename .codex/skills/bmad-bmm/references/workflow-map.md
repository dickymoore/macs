# BMM Workflow Map (summary from docs)

## Phase 1: Analysis (optional)
- `brainstorm` -> brainstorming report
- `research` -> research findings
- `create-product-brief` -> product-brief.md

## Phase 2: Planning
- `create-prd` -> PRD.md
- `create-ux-design` -> ux-spec.md

## Phase 3: Solutioning
- `create-architecture` -> architecture.md + ADRs
- `create-epics-and-stories` -> epic files + stories
- `check-implementation-readiness` -> PASS / CONCERNS / FAIL

## Phase 4: Implementation
- `sprint-planning` -> sprint-status.yaml
- `create-story` -> story-[slug].md
- `dev-story` -> code + tests
- `code-review` -> review report
- `correct-course` -> updated plan / re-route
- `retrospective` -> lessons learned

## Quick Flow (small work)
- `quick-spec` -> tech-spec.md
- `quick-dev` -> code + tests

## Brownfield
- `document-project` -> project-context.md (run before Phase 4 when docs are missing)
