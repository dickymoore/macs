# Epic 3 Retrospective

Date: 2026-04-10
Epic: 3
Epic Title: Route Work Safely and Reserve Protected Surfaces
Status: complete

## Epic Review

Epic 3 finished with all four stories complete and accepted:

- `3.1` configured workflow-aware routing policy.
- `3.2` recorded explainable assignment decisions.
- `3.3` reserved protected surfaces with coarse default locks.
- `3.4` blocked conflicts and duplicate ownership claims.

The sequencing held up. Routing policy came before rationale capture, which came before lock reservation and conflict blocking. That order kept controller-owned assignment truth explicit before protected-surface enforcement started rejecting unsafe work.

## What Worked

- Routing policy stayed centralized instead of drifting into adapter-specific heuristics.
- Assignment-decision recording made later task inspection and operator trust feasible.
- Coarse default locks were a pragmatic first boundary that solved real conflicts without overdesigning the lock model.
- Blocking duplicate ownership claims early protected the single-owner invariant before recovery work arrived.

## Friction and Risks

- Operator-facing list, inspect, and lifecycle command paths were still more fragmented than the controller model underneath them.
- Degraded evidence and worker-pane context were not yet surfaced where operators needed them.
- Intervention controls such as pause or accessibility-oriented terminal output were still missing from the main operational path.
